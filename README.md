# LISP-PL

This project is intended to create a functional Racket/LISP REPL.

In college, I wrote a C compiler in racket and thought it would be interesting to do the opposite.

The intention here is to avoid using external dependencies, simply building with rust's inbuilt functionality.

#### Why Racket? 
- I love LISP languages, I don't have much practical use for them in my day-to-day work, but want to play around with them occasionally.
